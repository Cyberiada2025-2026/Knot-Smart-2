## Contributing
Check out the [contribution guidelines](https://github.com/Cyberiada2025-2026/Knot-Smart/blob/new-readme/docs/contribution_guidelines.md) before making pull requests.

> [!IMPORTANT]
> **tl;dr**: Make your code feel as if it's part of the Godot standard library. If you can refactor something into a composition of nodes you should probably do it.

## Project structure
The project structure follows the style described in [the following article](https://rp.wtf/articles/organizing-files-in-godot/#a-more-complete-example).
